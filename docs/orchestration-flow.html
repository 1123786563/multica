1. 编排系统总览

当一个 Issue 被分配给 Agent 时，Orchestrator 自动创建一个 Plan（计划），其中包含三个有序 Node（节点）：Plan → Execute → Verify。每个 Node 通过 DAG 边声明依赖，仅在上游 Node 完成后才被调度。

6. 人工决策：Approve / RequestChanges / Retry

7. 自动重试与失败恢复

每个 Node 有 max_attempts（默认 2）。失败时若未耗尽重试次数，Orchestrator 自动重新调度，并携带 priorEvidenceSummary 和 changeRequest 帮助 Agent 避免重复错误。重试耗尽后自动创建 Attention Comment 通知用户。
