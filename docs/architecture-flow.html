1. 系统架构总览

Multica 采用 Go 后端 + pnpm monorepo 前端的架构。前端共享 packages 跨 Web 和 Desktop 复用，后端通过 Event Bus + WebSocket 实现实时推送。

支持邮箱验证码、Google OAuth、PAT（Personal Access Token）、Daemon Token 四种认证方式。Web 优先使用 HttpOnly Cookie，Desktop/Electron 使用 localStorage token。

5. Agent 任务执行流程

用户通过 Issue 或 Chat 触发 Agent 任务。后端创建 Task 并入队，Daemon 通过 WebSocket 唤醒 → HTTP claim → 执行 → 回报结果。全程 WS 事件驱动前端实时更新。

sequenceDiagram participant U as 用户 participant FE as 前端 participant API as Go Server participant Bus as Event Bus participant DH as Daemon Hub participant D as Daemon (Agent Runtime) participant DB as PostgreSQL Note over U,DB: 触发任务 U->>FE: 点击 "Run Agent" 或发送 Chat 消息 FE->>API: POST /api/issues/{id} (assign agent)
或 POST /api/chat/sessions/{id}/messages API->>DB: 创建 Task (status=queued) API->>Bus: Publish task:queued Bus->>DH: NotifyTaskAvailable(runtimeID, taskID) DH-->>D: WS: daemon:task_available Bus-->>FE: WS: task:queued (乐观更新) Note over U,DB: Daemon 认领 D->>API: POST /api/daemon/runtimes/{id}/tasks/claim API->>DB: UPDATE tasks SET status='running' API->>Bus: Publish task:dispatch Bus-->>FE: WS: task:dispatch (status→running) Note over U,DB: 执行与进度 D->>API: POST /api/daemon/tasks/{id}/start API->>Bus: Publish (→ task 生命周期) loop 流式输出 D->>API: POST /api/daemon/tasks/{id}/messages API->>Bus: Publish task:message Bus-->>FE: WS: task:message (setQueryData) end Note over U,DB: 完成/失败 alt 成功 D->>API: POST /api/daemon/tasks/{id}/complete API->>DB: UPDATE status='completed' API->>Bus: Publish task:completed Bus-->>FE: WS: task:completed else 失败 D->>API: POST /api/daemon/tasks/{id}/fail API->>DB: UPDATE status='failed' API->>Bus: Publish task:failed Bus-->>FE: WS: task:failed end

6. Daemon 通信流程

Daemon 是运行在用户本地的 Agent 运行时管理进程。它通过 HTTP + WebSocket 双通道与后端通信。WebSocket 用于即时唤醒和心跳，HTTP 用于可靠的任务 claim 和结果回报。
